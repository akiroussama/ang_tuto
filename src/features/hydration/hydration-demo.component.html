Incremental Hydration

Note: True Server-Side Rendering (SSR) and hydration cannot be executed in this Applet environment. This demo visually simulates and explains how the feature works in a full Angular Universal application.

Why it matters

  • Faster Time-to-Interactive (TTI): Instead of hydrating the entire application at once, Angular can hydrate components ("islands") incrementally. This makes the page interactive much faster, especially on mobile devices.
  • Prioritized Hydration: You can control the hydration strategy, e.g., hydrate components as they enter the viewport, on user interaction, or based on other triggers.
  • Improved Core Web Vitals: By reducing the main thread blocking time during initial load, incremental hydration significantly improves key performance metrics.

How to verify (in a real SSR app)

  1. The page would be served from the server as static HTML, loading instantly.
  2. Open the developer console. You would see a log for "Hydrating: WIDGET-A" as soon as the page loads, because it's visible.
  3. Scroll down. As the second widget enters the viewport, a new log "Hydrating: WIDGET-B" would appear.
  4. This proves that hydration is happening incrementally, only when needed.
