Why it matters

  • Performance: Updates only the parts of the DOM that depend on a signal's value. No more full component tree checks.
  • Simplicity: Eliminates the need for RxJS subjects for simple component state.
  • Clarity: computed() and effect() create a clear, reactive data flow that is easy to reason about.

How to verify

  1. Click "Increment" and watch the values update instantly.
  2. Observe the "Effect Logs" panel. A new log appears each time the count signal changes.
  3. Note that computed() values are memoized; they only re-calculate when a dependency changes, making them highly efficient.
